### [premium_gate.dart: baris 62] Login Google tidak ada error handling

**Severity:** 🔴 Critical
**File:** `lib/widgets/premium_gate.dart` (line 62)
**Problem:**
```dart
onPressed: () => _authService.signInWithGoogle(),
```
**Kenapa bermasalah:** Method `signInWithGoogle()` dipanggil tanpa try-catch. Jika terjadi error (misal: Google Sign-In belum dikonfigurasi, Firebase Auth Google provider belum di-enable, atau network error), exception akan thrown tanpa ditangkap.         SnackBar(content: Text('Gagal login: $e')),
```dart
      );
    }
  }
},
```

### [index.html: baris 19-31] Meta tag Google Sign-In client ID tidak ada untuk Flutter Web

**Severity:** 🔴 Critical
**File:** `web/index.html` (line 19-31)
**Problem:**
```html
<!-- Tidak ada meta tag google-signin-client_id -->
<meta charset="UTF-8">
<meta content="IE=Edge" http-equiv="X-UA-Compatible">
...
```
**Kenapa bermasalah:** Package `google_sign_in` di Flutter Web membutuhkan meta tag `<meta name="google-signin-client_id" content="YOUR_CLIENT_ID.apps.googleusercontent.com">` di `<head>` pada `index.html`. Tanpa ini, `GoogleSignIn().signIn()` akan selalu gagal di web tanpa error message yang jelas — menyebabkan tombol login "tidak response".
**Fix:**
```html
<head>
  <base href="$FLUTTER_BASE_HREF">

  <!-- Google Sign-In untuk Flutter Web -->
  <meta name="google-signin-client_id" content="373730537373-XXXXX.apps.googleusercontent.com">

  <meta charset="UTF-8">
  ...
```
> Ganti `373730537373-XXXXX.apps.googleusercontent.com` dengan OAuth 2.0 Web Client ID dari Google Cloud Console project `iconicapp-447005`.

**Problem:**

**Kenapa bermasalah:** Tidak ada loading state di `WeddingProvider`. Saat pertama kali app dibuka, `_plan` masih null (karena stream belum emit data), sehingga `hasPlan` = false dan langsung menampilkan `SetupScreen`. Jika koneksi lambat, user mungkin melihat SetupScreen sebelum data plan ter-load, lalu tiba-tiba berpindah ke AppShell — membingungkan user. **Fix:**

```dart
class WeddingProvider extends ChangeNotifier {
  bool _isLoading = true;
  bool get isLoading => _isLoading;

  void loadPlan(String planId) {
    _planId = planId;
    _isLoading = true;
    notifyListeners();

    _planSub = _service.getPlan(planId).listen((p) {
      _plan = p;
      _isLoading = false;
      notifyListeners();
    }, onError: (e) {
      _isLoading = false;
      _error = 'Gagal memuat: $e';
      notifyListeners();
    });
    // ... tasks & budget streams
  }
}
```
Dan di `AppShell`:
```dart

if (!provider.hasPlan) return const SetupScreen();
```

### [firebase_service.dart: baris 88] generateDefaultTasks menggunakan DateTime arithmetic yang bisa overflow

**Severity:** 🟢 Suggestion
**File:** `lib/services/firebase_service.dart` (line 88)
**Problem:**
```dart
final dueDate = phase == TaskPhase.week1
    ? weddingDate.subtract(const Duration(days: 7))
    : DateTime(weddingDate.year, weddingDate.month - phase.monthsBefore, weddingDate.day);
```
**Kenapa bermasalah:** Jika `weddingDate.month - phase.monthsBefore` menghasilkan angka ≤ 0 (misal: pernikahan bulan Januari dikurangi 12 bulan = bulan -11), Dart `DateTime` akan overflow ke tahun sebelumnya dengan benar, tapi jika `day` = 31 dan bulan target hanya punya 28/30 hari, tanggal akan roll over ke bulan berikutnya.
**Fix:**
```dart
DateTime _subtractMonths(DateTime date, int months) {
  var year = date.year;
  var month = date.month - months;
  while (month <= 0) {
    month += 12;
    year--;
  }
  final lastDay = DateTime(year, month + 1, 0).day;
  return DateTime(year, month, date.day.clamp(1, lastDay));
}
```
